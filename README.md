# مستند جامع پروژه کارت سرویس دیجیتال خودرو

## 1. معرفی پروژه

این پروژه یک سامانه تحت وب برای جایگزینی کارت های کاغذی تعویض روغنی ها با کارت دیجیتال مبتنی بر QR Code است.

در روش فعلی، تعویض روغنی ها معمولا پس از انجام سرویس، یک کارت کاغذی به مشتری می دهند که روی آن اطلاعاتی مثل تاریخ سرویس، کیلومتر فعلی، کیلومتر بعدی، نوع روغن و فیلترهای تعویض شده نوشته می شود. این کارت ها به راحتی گم می شوند، خراب می شوند، امکان پیگیری دقیق ندارند و برای صاحب تعویض روغنی هم ابزار مدیریتی یا تبلیغاتی ایجاد نمی کنند.

در این سامانه، به جای کارت کاغذی، یک QR Code روی خودرو یا در اختیار مشتری نصب/تحویل می شود. مشتری یا صاحب تعویض روغنی با اسکن این QR Code می توانند صفحه دیجیتال خودرو را مشاهده کنند. تعویض روغنی می تواند سرویس جدید ثبت کند و مشتری می تواند سوابق و زمان/کیلومتر پیشنهادی سرویس بعدی را ببیند.

## 2. هدف اصلی

هدف اصلی سامانه، دیجیتال کردن فرآیند ثبت، مشاهده و پیگیری سرویس های دوره ای خودرو برای تعویض روغنی ها و مشتریان است.

اهداف کلیدی:

- حذف کارت کاغذی و جایگزینی آن با QR Code اختصاصی.
- امکان مشاهده سوابق سرویس خودرو با موبایل.
- امکان ثبت سرویس جدید توسط تعویض روغنی.
- ایجاد پنل مدیریتی برای تعویض روغنی ها.
- ایجاد پنل مدیریتی برای ادمین سامانه.
- مدیریت اعتبار/کیف پول تعویض روغنی ها و کسر اعتبار به ازای ثبت سرویس.
- ارسال پیامک های سیستمی مثل رمز یکبارمصرف، تایید ثبت سرویس و اطلاع رسانی به مشتری.
- ایجاد صفحه دیجیتال تبلیغاتی برای هر تعویض روغنی.

### 5.2. کارت فعال

کارت فعال کارتی است که به یک خودرو و مشتری متصل شده و دارای سابقه سرویس است.

ویژگی ها:

- دارای سوابق سرویس.

### 5.3. کد اختصاصی تعویض روغنی

کد اختصاصی شناسه ای است که برای هر تعویض روغنی ایجاد می شود؛ مثل:

```text
SHOP_12345
```

کاربردها:

- شناسایی تعویض روغنی هنگام فعال سازی کارت.
- جلوگیری از فعال سازی کارت توسط افراد غیرمجاز.
- اتصال کارت ها و سرویس ها به تعویض روغنی مربوطه.

### 5.4. رمز یکبارمصرف

رمز یکبارمصرف برای عملیات حساس مثل فعال سازی کارت، ویرایش کارت و ثبت سرویس استفاده می شود.

قواعد پیشنهادی:

- رمز 4 یا 6 رقمی باشد.
- از طریق پیامک برای شماره موبایل ثبت شده تعویض روغنی ارسال شود.
- مدت اعتبار پیش فرض: 24 ساعت.
- سقف استفاده پیش فرض: 10 یا 20 عملیات، قابل تنظیم توسط ادمین.
- اگر مدت اعتبار یا تعداد استفاده تمام شود، رمز منقضی شود.
- درخواست رمز جدید، رمز قبلی را باطل کند.
- پس از ورود موفق، رمز می تواند در مرورگر همان دستگاه به صورت امن ذخیره شود تا تا زمان اعتبار، نیازی به ورود مجدد نباشد.
